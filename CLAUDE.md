# Gravel Race Automation — Claude Instructions

## Northstar
This repo serves the ecosystem master plan: **rated race databases (gravel/road/ski)
earn trust as honest race critics → convert to a plan/course/coaching ladder →
ultimately delivered on Endure Labs, replacing TrainingPeaks.** Gravel God is the
flagship traffic engine. When prioritizing, advance the current phase's KPI gate —
read `gravel-god-cycling/NORTHSTAR.md` (canonical plan, phases, anti-shill rules,
baseline metrics) before large or ambiguous work.

## Project Overview
Race database site (gravelgodcycling.com) — 757 race profiles, 543 tire guides, 154 VS pages, 60 state hubs, 1,538 total indexed URLs. Static HTML generated by Python scripts, deployed via tar+ssh to SiteGround WordPress hosting.

## Critical Rules

### HTML Generation
- **Never use innerHTML with data-derived values** — XSS risk. Use `textContent`/`createElement`. Tests enforce this (`test_no_innerhtml_with_data`).
- **Every generator must include `get_ga4_head_snippet()`** — pages without it are invisible to analytics. Preflight check `check_ga4_in_output_html()` validates this.
- **Race JSON structure**: `d['race']['gravel_god_rating']` — NOT `d.get('gravel_god_rating')`.
- **Score parsing**: Use `_parse_score()` — `int(raw)` crashes on `"3.5"`, `""`, `"None"`.
- **`json.dumps` inside `<script>` = injection** — `</script>` in data breaks HTML. Use `_safe_json_for_script()`.

### Email Capture Forms
- **Every email form MUST POST to its Cloudflare worker** (most capture surfaces use `https://fueling-lead-intake.gravelgodcoaching.workers.dev`; ratings use `review-intake`; see `workers/` for the full set — there is no generic `email-intake` worker). Never show a success message without actually submitting the email.
- **Always include honeypot field** — hidden input that bots fill, humans don't.
- **Use addEventListener, not inline onsubmit** — avoids JS injection via template variables in attribute context.

### A/B Testing
- **Experiment selectors must use `data-ab` attributes** — tests validate this.
- **Template selectors must reference existing DOM elements** — `test_experiment_templates.py` validates against a known-valid list.
- **Run experiments on race pages (`/race/*`)** — homepage/about don't get enough traffic for statistical significance.
- **Wildcard matching**: `gg-ab-tests.js` supports `/race/*` prefix matching.

### Analytics
- **Never fire GA events on auto-play/timers** — only manual user interactions.
- **GA4 key events require manual admin UI toggle** — see `docs/ga4-key-events.md`.
- **Custom dimensions for A/B reporting** must be registered in GA4 admin — see `docs/ga4-key-events.md`.

### Brand & Copy
- **Brand tokens source of truth**: `tokens.css`. Never hardcode hex.
- **Anti-slop rules**: `wordpress/slop_rules.py` — 27 banned phrases, 9 banned structures. Preflight warns on matches.
- **Neo-brutalist style**: No border-radius, no box-shadow, 2-3px solid borders.
- **Fonts**: Sometype Mono (data) + Source Serif 4 (editorial).

### Testing
- Run `python3 -m pytest tests/` — skip PIL-dependent tests with `--ignore` flags (6 test files need PIL).
- Run `python3 scripts/preflight_quality.py` before any deploy.
- `python3 wordpress/ab_experiments.py` must be run after changing experiments — `test_experiments_json_matches_source` enforces this.

### Deploy
- Race pages via tar+ssh (not rsync). Must use `{slug}/index.html` structure.
- **Articles** deploy via SCP to `/articles/{slug}/index.html` — NOT via `--sync-blog` (which deploys to `/blog/`).
- Always purge SiteGround cache after deploying.
- Full checklist: `python3 scripts/preflight.py --deploy`
- **Questionnaire page (`/questionnaire/`) is an ELEMENTOR page** (page ID
  5017, HTML widget `3f59420`) — NOT a Custom HTML block, despite the stale
  comment in `web/training-plans-questionnaire.html`. Editing `post_content`
  does nothing: Elementor renders `_elementor_data` and regenerates the
  post_content shadow on save. To change the form markup, edit the widget in
  the Elementor editor (or via its JS API: `$e.run('document/elements/settings', ...)`
  + `$e.run('document/save/default')`). The widget content has DRIFTED from
  the repo file (~30.4K vs ~32.8K) — repo is reference, widget is truth.
  Form JS deploys separately via `--sync-training` (SCP to wp-uploads).

### Articles (editorial content)
- Articles live in `wordpress/output/articles/`, NOT `wordpress/output/blog/`.
- Category `"article"` in blog-index.json. URL prefix `/articles/`, not `/blog/`.
- Classified by `robots` meta: `index, follow` = article, `noindex` = preview.
- Must include `@font-face` declarations (use `get_font_face_css()` from `brand_tokens.py`).
- Must include `get_ga4_head_snippet()` — test enforces via `test_article_infrastructure.py`.
- GA4 events: `article_scroll_depth` (all thresholds), `article_deep_read` (75%+ only), `article_cta_click`.
- Funnel report uses `article_deep_read`, NOT `article_scroll_depth` (which would inflate 4x).
- 54 tests in `test_article_infrastructure.py` guard against regressions.

## Known Pitfalls
1. Coaching carousel auto-advance was firing GA events every 6s — fixed, test enforces.
2. Blog generators (`generate_blog_preview.py`, `generate_blog_index_page.py`, `generate_race_recap.py`) are missing GA4 — not yet fixed.
3. 48 profiles have stale 2025 dates. 7 profiles have genuinely undateable TBD dates.
4. `esc(0)` is silently empty — Python falsiness. Check `is None or == ""`, not `if text`.
5. SVG attributes can't resolve `var()`/`color-mix()` — use CSS classes only.
6. **Articles in output/blog/ cause duplicate content.** `--sync-blog` deploys everything in `output/blog/` to `/blog/{slug}/`. Articles must live in `output/articles/` and deploy via SCP to `/articles/{slug}/`. A 301 redirect from `/blog/{slug}/` to `/articles/{slug}/` is in `.htaccess`.
7. **generate_blog_index.py classifier defaults to "preview"** if it can't detect category from slug pattern. Articles are detected via `robots` meta tag (`index, follow`). If the HTML is missing the robots tag, the article silently becomes a noindex preview.
8. **Font-face declarations are NOT optional.** Without `@font-face` blocks, the entire page renders in system fallback fonts (Georgia/monospace). Every new HTML page must include font loading from `brand_tokens.py`.
9. **Never edit hashed asset files directly.** `gg-search.{hash}.css` and `gg-styles.{hash}.css` have content hashes in filenames. If you edit the file without renaming, browsers serve stale cached CSS. Always rename with the new hash, or edit the source and re-extract.
10. **Inline event handlers (`onclick=`, `onsubmit=`) are banned.** Use `addEventListener`. Template variables in attribute context create injection risk. `test_ux_overhaul.py` enforces this — known exceptions tracked in `KNOWN_ONCLICK_LINES`.
11. **CSS `opacity: 0` on content sections causes invisible content.** Always use progressive enhancement: content visible by default, animations enabled only after JS adds a `.gg-animations-ready` class. `test_ux_overhaul.py::TestFadeSection` guards this.
12. **Shared header JS (`get_site_header_js()`) must be included in every generator.** It provides hamburger nav, sticky header, and publishes `--gg-header-height` CSS var. Without it, mobile nav doesn't work and sticky elements use fallback pixel values.
13. **URL params for search page filters must use full names.** `?discipline=`, `?tier=`, `?region=`, `?q=` — NOT abbreviated forms. `loadFromURL()` in `gravel-race-search.js` is the authority.
14. **When removing font files, also remove:** (a) `@font-face` declarations in `brand_tokens.py`, (b) CSS custom property tokens, (c) `<link rel="preload">` hints, (d) `FONT_FILES` list entries. `test_ux_overhaul.py::TestFontCleanup` enforces this.
15. **Cards with optional visual elements need CSS that works both with and without them.** Never change `padding: 16px` to `padding: 0` globally when adding thumbnails. Use a modifier class (`.gg-card-has-thumb`) to scope the override.
